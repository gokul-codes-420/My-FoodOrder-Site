1. Main Flow of the Website
	1.	Login Page (first page)
	2.	After login → Home Page
	3.	From Home → Menu (Indian Food Gallery) with 20+ food images
	4.	User adds items to Cart
	5.	User goes to Cart Page
	6.	User proceeds to Order / Checkout Page
	7.	User chooses Pay Online or Scan QR Code
	8.	After payment → Order Success / Confirmation Page

	•	“Sign Up” button → opens signup form:

7. Cart Page

When user clicks “Add to Cart”, the item goes into Cart and Cart count updates.

7.1 Cart Content Area

For each item:
	•	Small image (thumbnail)
	•	Name of dish
	•	Price per item
	•	Quantity selector: - 2 +
	•	Total for that item (Price × Quantity)
	•	Remove button (🗑)

7.2 Cart Summary Area

On side or bottom:
	•	Subtotal (sum of items)
	•	Delivery charge (e.g. ₹40)
	•	Taxes / GST (if applicable)
	•	Grand Total: ₹XXXX

Buttons:
	•	“Continue Shopping” → back to Menu
	•	“Proceed to Checkout” → go to Order / Payment section

The cart data must be saved until order is completed or cleared.

⸻

8. Order / Checkout Page

This is where user enters details and chooses payment.

8.1 Customer Details Form

Fields:
	•	Full Name
	•	Mobile Number
	•	Full Address
	•	City / Area (dropdown or text)
	•	Pincode
	•	Optional: “Delivery instructions” textbox

8.2 Order Summary

Show:
	•	List of items (like Cart, but read-only)
	•	Item total
	•	Delivery charge
	•	Final amount to pay

8.3 Payment Options

Show heading: “Choose Payment Method”

Options (radio buttons):
	•	( ) Pay Online (UPI/Card/NetBanking)
	•	( ) Scan QR Code (UPI)

If “Pay Online” is selected:
	•	Show “Pay Now” button
	•	This would connect to a payment gateway (Razorpay, Paytm, etc. – in real implementation).

If “Scan QR Code” is selected:
	•	Show QR Code image (your UPI or payment QR)
	•	Show steps:
	1.	Open your UPI app (GPay / PhonePe / Paytm etc.)
	2.	Scan the QR code
	3.	Pay the exact amount
	•	Optional field:
	•	“Enter Transaction ID / UPI Reference Number”
	•	Optional: upload screenshot (for manual verification)

8.4 Place Order Button

Button: “Place Order”

On click:
	•	Check → all form fields filled
	•	Check → payment method selected
	•	Save:
	•	User details
	•	Cart items
	•	Payment method (and transaction details, if QR)

Then redirect to Order Success Page.

⸻

9. Order Success / Confirmation Page

After a successful order:

Show:
	•	Big message: “Thank you! Your order has been placed.”
	•	Order ID (e.g. #ORD12345)
	•	Summary:
	•	Items ordered
	•	Total amount
	•	Payment method
	•	Delivery address
	•	Estimated delivery time (e.g. 30–45 mins)

Buttons:
	•	“Back to Home”
	•	“View My Orders” (if you implement order history)
